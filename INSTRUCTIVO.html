WebGenerica - Instructivo de uso

Que es WebGenerica

WebGenerica es una plataforma e-commerce lista para usar. Funciona como una tienda online completa que se puede personalizar para cualquier tipo de negocio sin tocar codigo. Todo se configura desde el panel de administracion.

La tienda incluye: catalogo de productos, carrito de compras, checkout con opciones de envio, sistema de cupones, paginas informativas (FAQ, Contacto, Quienes Somos, Servicio Tecnico) y un panel admin completo.

Panel de Administracion

Se accede desde el boton "Administrador" en el header (visible solo si estas logueado como admin).

Dashboard /admin

Resumen rapido con 3 tarjetas: total de productos, total de pedidos e ingresos totales.

Editar Inicio /admin/inicio

Controla todo lo que se ve en la pagina principal:

Carrusel Hero:

  • Agregar multiples slides (reordenables arrastrando)
  • Dos tipos: "Generico" (imagen de fondo) o "Producto destacado"
  • Imagen escritorio y celular separadas
  • Titulo, subtitulo, texto adicional
  • Tipografia: fuente Google, tamaño, peso, color
  • Layout: centrado, izquierda, derecha, esquinas
  • Overlay: color y opacidad sobre la imagen
  • Botones: texto, URL, estilo (relleno/borde), colores, reordenables

Beneficios: tarjetas debajo del hero con icono predefinido o personalizado, titulo y descripcion.

Seccion Instagram: titulo, URL del perfil, nombre de usuario. Se muestra como bloque con boton "Seguir".

Banner Promocional: imagen, titulo, subtitulo, URL y texto del enlace.

Productos /admin/productos

  • Crear, editar y eliminar productos
  • Campos: nombre, descripcion, precio, precio oferta, stock, categoria, destacado, activo
  • Subir multiples fotos (JPG/PNG/WebP, max 5MB, recomendado 800x800)
  • Reordenar fotos (la primera es la principal)
  • El slug se genera automaticamente del nombre

Categorias /admin/categorias

  • Crear, editar y eliminar categorias
  • Campos: nombre e imagen (por URL)
  • No se puede borrar una categoria con productos asociados

Pedidos /admin/pedidos

  • Lista de pedidos con datos del cliente, fecha, total y estado
  • Vista expandida: items, direccion de envio, metodo de pago
  • Cambiar estado: Pendiente, Procesando, Enviado, Entregado, Cancelado

Cupones /admin/cupones

  • Crear codigos de descuento (porcentaje o monto fijo)
  • Compra minima, usos maximos, fecha de expiracion
  • Activar/desactivar rapidamente desde la lista

FAQ /admin/faq

  • Titulo y subtitulo de la pagina
  • Lista de preguntas y respuestas, reordenables

Contacto /admin/contacto

  • Lista de telefonos, URL de Google Maps embebido, horarios de atencion

Quienes Somos /admin/nosotros

  • Titulo, subtitulo, descripcion larga, imagen, puntos destacados

Servicio Tecnico /admin/servicio

  • Hero: titulo y subtitulo
  • Tarjetas de servicios y beneficios con iconos
  • CTA WhatsApp: titulo, subtitulo, mensaje predeterminado

Configuracion General /admin/configuracion

  • Nombre del sitio (header, footer, titulo del navegador)
  • Logo (subir imagen)
  • Datos de contacto: telefono, email, direccion
  • Moneda: simbolo (ej: $, US$, €)
  • Redes sociales: Instagram, Facebook, Twitter/X, TikTok
  • Colores del tema: color primario y acento (selector de color)
  • Descripcion del footer
  • Opciones de envio: nombre, costo y descripcion

Como personalizar la tienda para un cliente nuevo

  1. Entrar al admin con admin@webgenerica.com / admin123
  2. Configuracion General: cambiar nombre, logo, colores, contacto, redes, moneda
  3. Editar Inicio: armar el carrusel hero con las imagenes del negocio, ajustar beneficios
  4. Categorias: crear las categorias del negocio
  5. Productos: cargar productos con fotos, precios y descripcion
  6. Paginas informativas: editar FAQ, Contacto, Quienes Somos, Servicio Tecnico
  7. Opciones de envio: configurar en la seccion de Configuracion
  8. Cupones: crear codigos de descuento si se necesitan

Con estos pasos, la tienda queda completamente adaptada al nuevo cliente sin tocar codigo.

Limitaciones actuales y soluciones a futuro

1. Imagenes se pierden al re-deployar el backend

Las imagenes se guardan en disco del servidor (Render Free). Cada nuevo deploy resetea el disco.

Solucion: Migrar a Cloudinary (gratis, 25GB) o Amazon S3.

2. Variantes de producto limitadas

El modelo soporta variantes (talle, color) pero el admin no permite crearlas.

Solucion: Agregar seccion de variantes al formulario de productos.

3. Imagenes de categorias por URL

Las categorias requieren pegar una URL en vez de subir archivo.

Solucion: Integrar el mismo componente de upload que usan los productos.

4. Sin gestion de pagos real

El checkout no procesa pagos reales, solo registra la orden.

Solucion: Integrar MercadoPago (ideal para Uruguay/Latam) o Stripe.

5. Sin notificaciones por email

El servicio de email esta configurado pero no envia emails automaticos.

Solucion: Configurar SMTP y activar emails transaccionales.

6. Backend se duerme (Render Free)

Se apaga tras 15 min sin uso. Primera visita tarda ~30 segundos.

Solucion: Upgrade a Render Starter ($7/mes) o usar UptimeRobot para pings.

7. Base de datos expira en 90 dias

PostgreSQL en Render Free tiene limite temporal.

Solucion: Migrar a Supabase o Neon (gratis permanente) antes de que expire.

8. Limite de 100 productos/pedidos en listados admin

Los listados cargan hasta 100 registros.

Solucion: Implementar paginacion en los listados del admin.

9. Sin sistema de roles granular

Solo hay dos roles: admin y customer.

Solucion: Agregar tabla de roles/permisos y middleware de autorizacion.

Flujo de deploy

Codigo local → git push → GitHub → Vercel (frontend) + Render (backend)

Cada push a la rama master dispara deploy automatico en ambas plataformas. No hay que hacer nada manual.
